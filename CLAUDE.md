# CLAUDE.md

## Project overview

`q` is a `-toolexec` preprocessor for Go ("Go wild with Q, the funkiest -toolexec preprocessor"). It originally landed as the question-mark / `try` operator (the seed proposal Go rejected) and has since grown a playground of helpers Go didn't ship: ctx cancellation checkpoints, futures and fan-in, panic→error recovery, mutex sugar, runtime preconditions, compile-time `dbg!`-style prints / slog attrs. Each `q.Try(call)` / `q.NotNil(p)` / `q.TryE(call).Method(…)` / `q.NotNilE(p).Method(…)` call site is rewritten at compile time into the conventional `if err != nil { return zero, … }` shape — so call sites read flat, but the generated code is identical to hand-written error forwarding with zero runtime overhead.

**Link gate via `_qLink`.** `pkg/q` declares a single bodyless `_qLink()` via `//go:linkname _qLink _q_atCompileTime` and forces its resolution with a package-level `var _ = _qLink`. `gopls`, `go vet`, and IDEs see ordinary Go — completion and analysis stay green. But `go build` / `go test` of any main or test target that imports `pkg/q` refuses to link without the preprocessor: the toolexec pass injects a no-op `_q_atCompileTime` companion file into `pkg/q`'s compile, so with `-toolexec=q` the build resolves and without it the linker fails on the missing symbol. Forgetting the preprocessor is a loud, deterministic error, never a silent loss of rewriting.

**Loud bodies for any rewriter miss.** Every public helper in `pkg/q` (`Try`, `NotNil`, `TryE`, `NotNilE`, plus all chain methods on `ErrResult[T]` / `NilResult[T]`) has a body that calls `panicUnrewritten("q.<name>")` and then returns the zero value. The preprocessor is supposed to rewrite every call site away before the user package is compiled; if any call survives the rewrite (rewriter bug, unsupported pattern), running it panics with a clear diagnostic naming the unrewritten helper. There is no silent fall-through to a "looks correct but discards the error" path.

**Why the bodies aren't bodyless.** Generic functions like `Try[T]` and methods on generic types like `ErrResult[T]` produce one mangled symbol per type instantiation. `//go:linkname` redirects a single local symbol to a single external one, so a fully bodyless generic declaration cannot be link-named to one universal stub the preprocessor supplies. The minimal-body shape (panic + return zero) is the closest equivalent.

## Current implementation state

- `pkg/q/q.go` — public surface:
  - `Try[T any](v T, err error) T` — bare error bubble.
  - `NotNil[T any](p *T) *T` — bare nil bubble using sentinel `q.ErrNil`.
  - `Ok[T any](v T, ok bool) T` — bare comma-ok bubble using sentinel `q.ErrNotOk`. Accepts either two separate args (`q.Ok(v, ok)`) or a single `(T, bool)`-returning call (`q.Ok(fn())`) via Go's f(g()) forwarding rule.
  - `Check(err error)` — void error-only bubble (for `db.Ping`, `file.Close`, `validate(x)`). Always an expression statement.
  - `Open[T any](v T, err error) OpenResult[T]` — resource acquisition; terminal methods are `.Release(cleanup ...func(T)) T` (variadic: zero args → preprocessor infers cleanup from T's type at compile time; one arg → explicit cleanup; two-or-more is a scanner diagnostic) and `.NoRelease() T` (same bubble path, no defer registered — opt-in "we acquired this and explicitly don't want to close it" form). Auto-Release supports channel types (`defer close(v)`), Close()-returning-error types (`defer func() { _ = v.Close() }()`, error discarded), and Close()-no-return types (`defer v.Close()`); the typecheck pass infers the kind via go/types and stores it on the qSubCall as AutoCleanup, with diagnostics for unrecognised types.
  - `TryE[T any](v T, err error) ErrResult[T]` — chain entry for custom error handling on the Try family.
  - `NotNilE[T any](p *T) NilResult[T]` — chain entry for custom nil handling on the NotNil family.
  - `OkE[T any](v T, ok bool) OkResult[T]` — chain entry for custom handling on the Ok family.
  - `CheckE(err error) CheckResult` — chain entry for custom error handling on Check (void).
  - `OpenE[T any](v T, err error) OpenResultE[T]` — chain entry for Open with shape methods that return `OpenResultE[T]` so `.Release` / `.NoRelease` can still terminate the chain.
  - `ErrResult[T]` methods: terminals `.Err(error)`, `.ErrF(func(error) error)`, `.Catch(func(error) (T, error))`, `.Wrap(string)`, `.Wrapf(string, ...any)`; chain-continuing recovers `.RecoverIs(sentinel error, value T) ErrResult[T]` (errors.Is match) and `.RecoverAs(typedNil error, value T) ErrResult[T]` (errors.As match — typedNil must be a typed-nil literal so the rewriter extracts the type at compile time). Recover steps must be followed by a terminal; standalone use is rejected.
  - `NilResult[T]` methods: `.Err(error)`, `.ErrF(func() error)`, `.Catch(func() (*T, error))`, `.Wrap(string)`, `.Wrapf(string, ...any)`.
  - `OkResult[T]` methods: `.Err(error)`, `.ErrF(func() error)`, `.Catch(func() (T, error))`, `.Wrap(string)`, `.Wrapf(string, ...any)` — mirrors NilResult's vocabulary (no captured source error on the not-ok branch, so Wrap uses `errors.New` and Wrapf uses `fmt.Errorf` without `%w`).
  - `CheckResult` methods (all return void): `.Err(error)`, `.ErrF(func(error) error)`, `.Catch(func(error) error)` (nil suppresses, non-nil bubbles), `.Wrap(string)`, `.Wrapf(string, ...any)`.
  - `OpenResultE[T]` shape methods: `.Err(error)`, `.ErrF(func(error) error)`, `.Catch(func(error) (T, error))`, `.Wrap(string)`, `.Wrapf(string, ...any)` — each returns `OpenResultE[T]` so `.Release(cleanup func(T)) T` or `.NoRelease() T` can follow as terminal.
  - `Trace[T any](v T, err error) T` — Try-shape with call-site `file:line` prefix injected into the bubble at compile time.
  - `TraceE[T any](v T, err error) TraceResult[T]` — chain variant; every method composes over the location prefix.
  - `Recv[T any](ch <-chan T) T` — channel receive that bubbles `q.ErrChanClosed` on closed channel.
  - `RecvE[T any](ch <-chan T) OkResult[T]` — chain variant (reuses OkResult).
  - `As[T any](x any) T` — type assertion that bubbles `q.ErrBadTypeAssert` on failure. Explicit type arg required (`q.As[User](x)`).
  - `AsE[T any](x any) OkResult[T]` — chain variant.
  - `Lock(l sync.Locker)` — statement-only; emits `_qLockN := l; _qLockN.Lock(); defer _qLockN.Unlock()`.
  - `TODO(msg ...string)` / `Unreachable(msg ...string)` — statement-only panics with file:line-prefixed messages.
  - `Require(cond bool, msg ...string)` — statement-only; bubbles `fmt.Errorf("<file>:<line>[: <msg>]: %w", q.ErrRequireFailed)` to the enclosing function's error return when cond is false. The %w-wrapped sentinel makes `errors.Is(err, q.ErrRequireFailed)` work across the wrap. Replaces the previous panic-shaped `q.Assert`; q's stance is that the library produces errors, not panics.
  - `DebugPrintln[T any](v T) T` — panic stub; the preprocessor rewrites each call site to `q.DebugPrintlnAt("<file>:<line> <src-text>", v)`. Pass-through, usable mid-expression. Renamed from the original `q.Debug`.
  - `DebugPrintlnAt[T any](label string, v T) T` — runtime helper paired with DebugPrintln. Writes to `q.DebugWriter` (default `os.Stderr`, reassignable for test capture).
  - `DebugSlogAttr[T any](v T) slog.Attr` — panic stub; the preprocessor rewrites each call site directly to `slog.Any("<file>:<line> <src-text>", v)`. No q runtime helper on the value path — expands to stdlib `log/slog`. The rewriter detects this family at the shape level and injects the `log/slog` import.
  - `SlogAttr[T any](v T) slog.Attr` — panic stub; rewrites to `slog.Any("<src-text>", v)` (no file:line prefix — the production-grade slog helper, vs DebugSlogAttr's dev-time flavor).
  - `SlogFile() slog.Attr` — panic stub; rewrites to `slog.Any("file", "<basename>")` from the call site's source position.
  - `SlogLine() slog.Attr` — panic stub; rewrites to `slog.Any("line", <line>)` from the call site's source position.
  - `SlogFileLine() slog.Attr` — panic stub; rewrites to `slog.Any("file", "<basename>:<line>")` (combined location string).
  - `File() string` / `Line() int` / `FileLine() string` — panic stubs; rewrite to plain Go literals (`"main.go"`, `42`, `"main.go:42"`) at compile time.
  - `Expr[T any](v T) string` — panic stub; rewrites to `"<source-text-of-v>"` (literal string of the argument's source text). The argument's runtime value is discarded — only its source spelling is captured. Generic `T any` so any expression form is accepted.
  - `EnumValues[T comparable]() []T` / `EnumNames[T comparable]() []string` — panic stubs; rewrite to a literal slice of T's package-scope const values / their identifier names in source declaration order. Constants are discovered by the typecheck pass walking T's declaring package for `*types.Const` whose type matches T. Same-package T only — cross-package T surfaces a diagnostic asking for a thin local wrapper.
  - `EnumName[T comparable](v T) string` / `EnumValid[T comparable](v T) bool` / `EnumOrdinal[T comparable](v T) int` / `EnumParse[T comparable](s string) (T, error)` — panic stubs; rewrite to inline IIFE-wrapped switches over T's constants. EnumName returns `""` on miss, EnumValid returns false, EnumOrdinal returns -1, EnumParse returns `(zero, fmt.Errorf("%q: %w", s, q.ErrEnumUnknown))`. NAME-based: the input/output uses the constant identifier (so `q.EnumParse[Status]("Done")` returns `Done`, not `q.EnumParse[Status]("done")`). Works for both int- and string-backed enums (any `comparable` const-able type).
  - `ErrEnumUnknown` sentinel, wrapped via `%w` into the EnumParse bubble for unknown names.
  - In-place q.* calls are accepted in container-statement headers (`IfStmt.Init` / `Cond`, `ForStmt.Init` / `Cond` / `Post`, `RangeStmt.X`, `SwitchStmt.Init` / `Tag`). Each q.* call gets its own synthetic-ExprStmt-wrapped shape so the edit span equals the call's `OuterCall.Pos/End` — without per-call scoping the container-level edit would overlap with edits the body's own statements emit. Bubble-shape q.* in the same positions surfaces a diagnostic asking the user to extract the call to a preceding statement (a multi-line bind+check would break the header's single-line parse).
  - The `//line` directive injection (rewriteFile) is conditional on the rewrite actually adding lines — counted via `bytes.Count(src[start:end], "\n")` vs `strings.Count(text, "\n")`. Span-substituting in-place rewrites preserve line counts, so no directive is added; their downstream-line mapping stays correct without one. Bubble rewrites still get the directive. Avoiding the directive is required for in-place rewrites inside container headers, where a stray `\n//line\n` would split the header invalidly.
  - `Exhaustive[T any](v T) T` — panic stub; legal ONLY as the tag of a switch statement (`switch q.Exhaustive(v) { case … }`). The typecheck pass walks T's declaring package for `*types.Const` of T and validates that every constant appears in some case clause; missing constants abort the build with a diagnostic. A `default:` clause does NOT replace coverage of declared constants — it catches values outside the declared set (forward-compat / Lax-JSON-opted types / runtime drift), and the declared constants still each need a dedicated case. Supports multi-value cases (`case A, B:`) and switch-with-init. Cross-package T is rejected — case clauses would need qualified names that the rewriter doesn't yet emit. Found anywhere except a switch tag, the scanner emits a diagnostic. The wrapper is removed at rewrite time, leaving a plain `switch v { … }` (zero runtime overhead).
  - `F(format string) string` — panic stub; rewrites to `fmt.Sprintf` with `{expr}` placeholders extracted as positional `%v` args. The format MUST be a Go string literal (validated at scan time). Brace-escape `{{` / `}}` for literal braces. Inside a placeholder, Go string and rune literals are honoured so braces inside them don't terminate the placeholder. The extracted expressions are validated via `parser.ParseExpr`; malformed placeholders abort the build with a diagnostic naming the offending text.
  - `Ferr(format string) error` — panic stub; rewrites to `errors.New(<format>)` (no exprs) or `fmt.Errorf(<format>, <exprs>...)` (with exprs). Same brace semantics as `q.F`. Produces a fresh error — does not wrap.
  - `Fln(format string)` — panic stub; rewrites to `fmt.Fprintln(q.DebugWriter, <message>)`. Routes through `DebugWriter` so test fixtures can capture output, same destination as `q.DebugPrintln`.
  - `SQL(format string) SQLQuery` / `PgSQL(format string) SQLQuery` / `NamedSQL(format string) SQLQuery` — panic stubs; rewrite to a `q.SQLQuery` composite literal whose `Query` carries placeholder-style SQL (`?` for SQL, `$1`/`$2` for PgSQL, `:name1`/`:name2` for NamedSQL) and whose `Args []any` carries the extracted Go expressions in left-to-right order. The format MUST be a Go string literal (validated at scan time — accepting dynamic formats would re-open the SQL-injection hole the helper exists to close). Same `{expr}` / `{{` / `}}` syntax as `q.F`. The `%` character is NOT escaped (unlike q.F) — fmt.Sprintf is not on the output path, so `%` is a plain SQL character.
  - `SQLQuery struct { Query string; Args []any }` — real (non-stub) type that pairs the parameterised query with its argument list. Drop directly into a driver: `db.QueryRowContext(ctx, s.Query, s.Args...)`.
  - `GenStringer[T comparable]() GenMarker` / `GenEnumJSONStrict[T comparable]() GenMarker` / `GenEnumJSONLax[T comparable]() GenMarker` — package-level "directives" written as `var _ = q.GenX[T]()`. The scanner detects these top-level var declarations, the typecheck pass resolves T's constants (reusing resolveEnum's mechanism), and a separate file-synthesis pass emits a `_q_gen.go` companion file (per-package, written to $TMPDIR) containing the requested method declarations. The rewriter additionally substitutes each directive call with `q.GenMarker{}` so the var has a no-op runtime initializer. `GenStringer` synthesizes `func (T) String() string` (switch over names, `""` on unknown). `GenEnumJSONStrict` synthesizes name-based `MarshalJSON`/`UnmarshalJSON` that error on unknown values (compatible with q.Exhaustive — every successfully-unmarshalled value is a declared constant). `GenEnumJSONLax` synthesizes pass-through marshallers using T's underlying type — unknown wire values survive round-trip unchanged (forward-compat).
  - `GenMarker struct{}` — the value type the Gen* directives return. No runtime data; only purpose is to keep `var _ = q.GenX[T]()` type-checkable.
  - `Fields[T any]() []string` / `AllFields[T any]() []string` — panic stubs; rewrite to a literal `[]string{...}` of T's struct field names. Fields skips unexported; AllFields includes them. Both follow pointer indirection (`*User` works the same as `User`). The typecheck pass walks the struct's `*types.Struct` fields and populates `qSubCall.StructFields` (in source declaration order). Non-struct T surfaces a diagnostic.
  - `TypeName[T any]() string` — panic stub; rewrites to a string literal carrying T's defined type name. Pointer types dereference (`*User` → `"User"`); unnamed types use `types.TypeString` with the short package qualifier.
  - `Tag[T any](field, key string) string` — panic stub; rewrites to a string literal carrying the struct-tag value for the given field+key on T. Both `field` and `key` MUST be Go string literals (validated at scan time). The typecheck pass uses an embedded `reflect.StructTag.Get`-equivalent parser (`reflectStructTag.Get` in `typecheck.go`) so the rewriter doesn't pull `reflect` as a dep. Field-not-found surfaces a diagnostic.
  - `Upper(s string) string` / `Lower(s string) string` / `Snake(s string) string` / `Kebab(s string) string` / `Camel(s string) string` / `Pascal(s string) string` / `Title(s string) string` — panic stubs; rewrite at compile time to a Go string literal carrying the case-transformed text. Argument MUST be a Go string literal (validated at scan time). Tokenisation handles camelCase, PascalCase, kebab-case, snake_case, space-separated, and acronym runs (`XMLHttpRequest` → `xml_http_request`); see `internal/preprocessor/strings.go` for the splitWords / joinWords helpers.
  - `SlogCtx(ctx context.Context, attrs ...slog.Attr) context.Context` — runtime helper (NOT rewritten); appends slog.Attrs to a private ctx key. Repeat calls accumulate.
  - `SlogContextHandler(base slog.Handler) slog.Handler` — runtime helper; wraps base so every Handle call attaches ctx-accumulated attrs to the record before forwarding.
  - `InstallSlog(base slog.Handler)` — runtime helper; calls `slog.SetDefault(slog.New(SlogContextHandler(base)))`.
  - `InstallSlogJSON(w io.Writer, opts *slog.HandlerOptions)` / `InstallSlogText(w, opts)` — sugar over InstallSlog with stdlib JSON / text base handlers (w defaults to os.Stderr).
  - `DebugWriter io.Writer = os.Stderr` — user-reassignable destination for DebugPrintlnAt output.
  - `Async[T any](fn func() (T, error)) Future[T]` — runtime helper; spawns `fn` in a goroutine, returns a `Future[T]` backed by a buffered channel.
  - `AwaitRaw[T any](f Future[T]) (T, error)` — runtime helper; blocks on the Future and returns the tuple.
  - `Await[T any](f Future[T]) T` — Try-like bubble over `q.AwaitRaw(f)`.
  - `AwaitE[T any](f Future[T]) ErrResult[T]` — chain variant; reuses `ErrResult` so the vocabulary matches TryE.
  - `CheckCtx(ctx context.Context)` — statement-only ctx.Err() checkpoint (covers both cancellation and deadline expiration since ctx.Err() returns context.Canceled or context.DeadlineExceeded); rewrites to `_qErrN := (ctx).Err(); if _qErrN != nil { return zero, _qErrN }`. Renamed from the original q.Bubble.
  - `CheckCtxE(ctx context.Context) CheckResult` — chain variant; reuses `CheckResult` vocabulary (Err/ErrF/Wrap/Wrapf/Catch); always statement-only.
  - `RecvRawCtx[T any](ctx, ch) (T, error)` — runtime helper (NOT rewritten); `select` on ctx.Done + ch; returns `(zero, ctx.Err())` on cancel, `(zero, ErrChanClosed)` on close, `(v, nil)` on receive.
  - `RecvCtx[T any](ctx, ch) T` — Try-like bubble over `q.RecvRawCtx(ctx, ch)`.
  - `RecvCtxE[T any](ctx, ch) ErrResult[T]` — chain variant; reuses `ErrResult`.
  - `AwaitRawCtx[T any](ctx, f) (T, error)` — runtime helper (NOT rewritten); `select` on ctx.Done + Future channel.
  - `AwaitCtx[T any](ctx, f) T` / `AwaitCtxE[T any](ctx, f) ErrResult[T]` — rewritten like Await / AwaitE but inner text is `q.AwaitRawCtx(ctx, f)`.
  - `Timeout(ctx context.Context, dur time.Duration) context.Context` — rewritten in define/assign position only; emits `<lhs>, _qCancelN := context.WithTimeout(ctx, dur); defer _qCancelN()`. Auto-injects `context` import. Other forms (hoist/return/discard) are diagnostics — the `defer cancel()` has no home.
  - `Deadline(ctx context.Context, t time.Time) context.Context` — same shape as Timeout but emits `context.WithDeadline`.
  - `AwaitAllRaw[T any](futures ...Future[T]) ([]T, error)` — runtime helper (NOT rewritten); spawns one collector goroutine per future, gathers in input order, bubbles first error.
  - `AwaitAllRawCtx[T any](ctx, futures ...Future[T]) ([]T, error)` — same with outer `select` on ctx.Done — returns `ctx.Err()` once, not an N-copy aggregate.
  - `AwaitAll[T any](futures ...Future[T]) []T` — rewritten; Try-like bubble over `q.AwaitAllRaw`. Supports variadic spread (`fs...`) via `EntryEllipsis` on `qSubCall`.
  - `AwaitAllE[T any](futures ...Future[T]) ErrResult[[]T]` — chain variant.
  - `AwaitAllCtx[T any](ctx, futures ...Future[T]) []T` / `AwaitAllCtxE` — ctx-aware; inner is `q.AwaitAllRawCtx`.
  - `AwaitAnyRaw[T any](futures ...Future[T]) (T, error)` — runtime helper; returns first-success, else `errors.Join(...)` of all per-future errors in completion order.
  - `AwaitAnyRawCtx[T any](ctx, futures ...Future[T]) (T, error)` — same with outer `select` on ctx.Done.
  - `AwaitAny[T any](futures ...Future[T]) T` / `AwaitAnyE` / `AwaitAnyCtx` / `AwaitAnyCtxE` — rewritten variants, symmetrical to AwaitAll.
  - `RecvAnyRaw[T any](chans ...<-chan T) (T, error)` — runtime helper; dynamic N-way `reflect.Select` across channels, bubbles `ErrChanClosed` on any close.
  - `RecvAnyRawCtx[T any](ctx, chans ...<-chan T) (T, error)` — same with an extra ctx.Done case in the Select.
  - `RecvAny[T any](chans ...<-chan T) T` / `RecvAnyE` / `RecvAnyCtx` / `RecvAnyCtxE` — rewritten; Try-like bubble over the corresponding runtime helper.
  - `Drain[T any](ch <-chan T) []T` — runtime helper (NOT rewritten); for-range until close, returns collected values. No error path.
  - `DrainAll[T any](chans ...<-chan T) [][]T` — runtime helper; one goroutine per channel drains until close, results in input order. `sync.WaitGroup`-based.
  - `DrainRawCtx[T any](ctx, ch <-chan T) ([]T, error)` — runtime helper; `select`-based for-range; returns `(nil, ctx.Err())` on cancel.
  - `DrainAllRawCtx[T any](ctx, chans ...<-chan T) ([][]T, error)` — same per-channel fan-out with outer `select` on ctx.Done.
  - `DrainCtx[T any](ctx, ch) []T` / `DrainCtxE` — rewritten; Try-like bubble over DrainRawCtx.
  - `DrainAllCtx[T any](ctx, chans ...<-chan T) [][]T` / `DrainAllCtxE` — rewritten; bubble over DrainAllRawCtx.
  - No `DrainE` / `DrainAllE` non-ctx forms exist — without ctx, Drain can't fail, so the E chain has nothing to shape. Skipped deliberately.
  - No `DrainAny` — ambiguous semantics; handwritten composition is clearer. Skipped deliberately.
  - No `RecvAll` — would confuse with Drain. Skipped deliberately.
  - `Recover(errPtr *error)` — runtime helper; `defer q.Recover(&err)` converts any panic into `*q.PanicError` assigned via errPtr.
  - `RecoverE(errPtr *error) RecoverResult` — chain variant; methods `.Map(func(any) error)`, `.Err(error)`, `.ErrF(func(*PanicError) error)`, `.Wrap(string)`, `.Wrapf(string, ...any)`. All terminal (each is the deferred function; recover() sees the panic).
  - `PanicError{Value any; Stack []byte}` — `error` type produced by Recover / RecoverE (except when Err/Map overrides). `errors.As` extracts it.
  - `ErrNil` sentinel, exposed for `errors.Is` checks against the bare `q.NotNil` bubble.
  - `ErrNotOk` sentinel, exposed for `errors.Is` checks against the bare `q.Ok` bubble.
  - `ErrChanClosed` sentinel, exposed for `errors.Is` checks against the bare `q.Recv` bubble.
  - `ErrBadTypeAssert` sentinel, exposed for `errors.Is` checks against the bare `q.As` bubble.
  - `ErrRequireFailed` sentinel, wrapped via `%w` into the `q.Require` bubble; `errors.Is(err, q.ErrRequireFailed)` identifies the failure mode regardless of the prefixed file:line / msg text.
  - `Const[T any](v T) func(error) (T, error)` — pure runtime helper (NOT rewritten); returns a Catch handler that always recovers to v. Pairs with `q.TryE/OpenE/TraceE/AwaitE.Catch(q.Const(0))` for a one-line "default value on any error" recovery.
  - `qRuntimeHelpers` carve-out (scanner): `ToErr`, `Const`, `DebugPrintlnAt`, `SlogCtx`, `SlogContextHandler`, `InstallSlog`, `InstallSlogJSON`, `InstallSlogText`, `Async`, `AwaitRaw`, `AwaitRawCtx`, `AwaitAllRaw`, `AwaitAllRawCtx`, `AwaitAnyRaw`, `AwaitAnyRawCtx`, `RecvRawCtx`, `RecvAnyRaw`, `RecvAnyRawCtx`, `Drain`, `DrainAll`, `DrainRawCtx`, `DrainAllRawCtx`, `Recover`, `RecoverE`. These are NOT rewritten and run at runtime.
  - `ToErr[T any, E any, P interface{ *E; error }](v T, e P) (T, error)` — runtime helper (NOT rewritten, real body executes) that adapts a `(T, *E)` callee into `(T, error)` while collapsing a typed-nil `*E` into a literal nil `error`. Satisfies the typed-nil guard (its return type is literally `error`). Usable standalone, not tied to the rest of q.
  - `GoroutineID() uint64` — exposes runtime `g.goid` via toolexec runtime injection. The preprocessor routes runtime-package compiles to `planRuntimeStub`, which appends a synthesized file declaring `runtime.GoroutineID() uint64 { return getg().goid }` with a single-arg `//go:linkname GoroutineID` directive. The single-arg form marks the symbol as intentionally externally-linkable, satisfying Go 1.23+'s third-party-into-runtime restriction. pkg/q's `GoroutineID` is a one-liner with `//go:linkname runtimeGoroutineID runtime.GoroutineID` pulling the symbol. Cost: ~1ns, an inlined struct-field read. NOT in `qRuntimeHelpers` carve-out (it's a regular pkg/q public function — only the runtime side gets injected). The runtime injection is the second package q messes with after pkg/q itself; same cache-discipline applies (a `runtime.a` from a non-toolexec build doesn't have GoroutineID and would fail to link if reused for a toolexec build).
  - `_qLink` plus `func init() { _qLink() }` — the package-level link gate.
- `cmd/q/main.go` — toolexec shim, thin wrapper around `internal/preprocessor.Run`.
- `internal/preprocessor/run.go` — toolexec entry: dispatch by tool, plan, forward.

Diagnostics from unsupported `q.*` shapes or typed-nil-interface hazards abort the build.
- `internal/preprocessor/typecheck.go` — error-slot type guard. Runs once per user-package compile using `go/types` with an importer backed by the compile's `-importcfg` (archives resolved via `importer.ForCompiler("gc", lookup)`). For every recognised `q.*` call site, checks the error-position type is exactly the built-in `error` interface; anything else (`*MyErr`, custom named types) produces a `file:line:col: q: …` diagnostic and aborts the build. Rationale: Go's `f(g())` forwarding rule plus implicit concrete-to-interface assignability means `func Foo() (T, *MyErr)` passed to `q.Try(Foo())` type-checks, but a nil `*MyErr` becomes a *non-nil* `error` value — the rewritten bubble check would fire for a notionally-nil error. This is [100 Go Mistakes #45](https://100go.co/#returning-a-nil-receiver-45). Best-effort: if the importcfg is missing or the importer can't be built, the guard silently skips (the real compile surfaces whatever fails). Per-family error-slot positions: Try/TryE/Open/OpenE → last tuple element of the inner call; Check/CheckE → the single argument's type. Fix paths for a rejected build: change the callee to return `error`, use `q.ToErr` to adapt, or convert explicitly at the call site. Also resolves enum constants for the `q.Enum*` families: walks T's declaring package for `*types.Const` whose type is identical to T, sorts by source position, populates `EnumConsts` + `EnumTypeText` on each call's qSubCall. Cross-package T surfaces a diagnostic.

All produce single self-contained expressions that drop into any expression position — no bind / bubble plumbing.

- `internal/preprocessor/scanner.go` — recognises `q.*` call expressions on the AST. Also maintains `qRuntimeHelpers` (currently `{"ToErr"}`): these q-aliased names are non-rewritten runtime helpers with real bodies, so `qCallRootPos` returns `NoPos` for calls whose entry name is in that set. Without this carve-out the "unsupported q.* shape" safety net would false-positive on standalone `q.ToErr(...)` usage. **Matches all four entries × five statement forms:** define (`v := …`), assign (`v = …`, including non-ident LHS like `obj.field`), discard (no-LHS expression statement), return (`return …, q.Try(…), …`), and hoist (`v := f(q.Try(…), q.NotNil(p))` — q.* nested anywhere inside any non-return statement's expressions). Each entry has a bare and chain (`.Err`/`.ErrF`/`.Catch`/`.Wrap`/`.Wrapf`) variant. Resolves the local import alias of `pkg/q` per file. Any unmatched `q.*` reference in a user file becomes a diagnostic. **Per-statement** fields live on `callShape` (Stmt, Form, LHSExpr, EnclosingFuncType); **per-call** fields live on `qSubCall` (Family, Method, MethodArgs, InnerExpr, OuterCall). A callShape owns `Calls []qSubCall` — length 1 for direct-bind forms, 1..N otherwise. `matchStatement` first tries the tight direct-bind path (single LHS, single RHS, RHS IS a q.* call with no nested q.*s); if eligibility fails, `matchHoist` / the return case collect every q.* in the statement's expressions via `collectQCalls` (which descends through matched q.*s too, so nested q.* inside another q.*'s InnerExpr gets caught). Descent stops at `*ast.FuncLit` boundaries (closures are scanned with their own `FuncType` via `walkFuncLits`).
- `internal/preprocessor/rewriter.go` — emits replacement source for one matched statement. `renderShape` orders `sh.Calls` innermost-first via `orderInnermostFirst` (so an outer q.*'s bind line can embed the inner's `_qTmpN`), allocates `_qTmpN` counters in that order so the generated code reads top-to-bottom with increasing N, then renders each sub-call's bind + bubble block and joins them with a newline + indent. `substituteSpans` is the universal "replace immediate-child q.* spans with their temps" primitive: used for each sub-call's InnerExpr text (via `commonRenderInputs` → `exprTextSubst`), for MethodArg text (Wrap/Wrapf/Err/ErrF/Catch args), and for the full-stmt reconstruction in `finalStmtSuffix`. "Immediate child" means contained in the target range but not contained by any other sub also in the range — so a great-grandchild only gets substituted once, by its direct parent's span. Per-family bind line still varies by form (`tryBindLine`, `nilBindLineAndCheck`); formReturn and formHoist both bind to `_qTmp<N>` regardless of family. Try family bubbles on `err != nil`; NotNil bubbles on `<checkVar> == nil`. The discard form binds to a fresh `_qVal<N>` for NotNil (no LHS to reuse) and `_, _qErr<N>` for Try. Bare and most chain methods share family-specific skeletons (`assembleErrBlock` / `assembleNilBlock`); `Catch` has its own assembler in each family because the err / nil branch may either recover (rebind the LHS via fn) or transform (bubble a new error). For discard form, Catch's recovery position becomes `_` since there's nowhere to put a recovered value. NotNil's `Wrap`/`Wrapf` differ from TryE's: there's no source error to %w-wrap, so they emit `errors.New(msg)` / `fmt.Errorf(format, args...)` respectively. Uses the universal `*new(T)` zero-value form. When a render needs `fmt` or `errors`, the rewriter calls `ensureImport` to inject the package into the file's import block (parenthesised form preferred; single-line and zero-import cases handled separately). Always appends a `var _ = <alias>.ErrNil` sentinel so the q import stays alive after rewrites erase the only callers.

### Implementation notes from the rewriter pass

- **Universal `*new(T)` zero values.** Avoids per-type knowledge of the spelling of zero (`0` for ints, `""` for strings, `nil` for pointers, `T{}` for structs, generic-aware variants, etc.). The compiler folds `*new(T)` to a constant zero — generated machine code is identical to a hand-written zero literal.
- **Sentinel reference to keep the q import alive.** Appending `var _ = <alias>.ErrNil` after every rewritten file is cheaper than statically tracking residual usage. Good enough until / unless gofmt-quality output of the temp files becomes interesting.
- **Link-gate construction matters.** The first attempt (`var _ = _qLink` at package level) compiled fine but was dead-code-eliminated by Go's optimiser — the linker then dropped `_qLink` as unreferenced and the gate silently disengaged (build succeeded without `-toolexec=q`). Switched to `func init() { _qLink() }` which calls the function explicitly and survives every optimisation level. `TestLinkGateFailsWithoutPreprocessor` is the regression guard.

## Conventions

- **Golden rule: only accept Go-valid syntax.** The surface q exposes to users must parse and type-check as plain Go — what `go build` / `gopls` / the IDE's analyzer sees before the toolexec pass. Never design a feature that relies on syntax Go itself would reject (e.g. auto-inferring a trailing `, nil` in a return with fewer values than the signature requires, or omitting an argument). That would produce red squiggles in every editor, defeating the whole reason we chose a toolexec rewriter over a custom parser: IDE integration. If a proposal requires Go to accept something it doesn't, reject the proposal — no matter how ergonomic the post-rewrite code would look.
- The preprocessor's job is narrow: scan bodies for `q.*` call expressions, pattern-match the chain shape (`q.TryE(call).Method(args)` is one expression, not two), build the inlined replacement using the enclosing function's return-type list, write the rewritten file to a tempdir, and substitute it into the compile's argv. No type-level algebra, no SMT.
- **Parse, don't template.** Every pass works from the AST of the source files the Go toolchain handed the compiler (`go/parser`, `go/ast`, `go/printer`); shape follows `proven` and `rewire`. Synthesized files go to `$TMPDIR` and join the compile argv. The on-disk source tree is never modified. Hardcoded textual templates that duplicate what is already in source are a smell — derive from the AST so API evolution in `pkg/q` flows through mechanically.
- Test via the e2e fixture harness, not ad-hoc shell scripts. Each new rewriter pattern gets a fixture under `internal/preprocessor/testdata/cases/<name>/`.

## Developing the preprocessor

**Cache discipline.** Go's build cache key does not include the toolexec binary's effect on source. A cached `pkg/q.a` from a plain `go test` (no stub) and one from a `-toolexec=q` build (with stub) have the same key — whichever ran first wins. Symptoms:

- `relocation target _q_atCompileTime not defined` — a toolexec build reused a stub-less artifact from an earlier non-toolexec run.
- A negative link-gate test (expected to fail without the preprocessor) silently succeeds — a non-toolexec build reused a stub-containing artifact.

Two protections in place so this rarely matters:

1. The e2e harness sets `GOCACHE` to a harness-owned tempdir. Tests under `go test ./...` and `go test ./internal/preprocessor/` do not cross-contaminate.
2. `TestLinkGateFailsWithoutPreprocessor` allocates *its own* tempdir cache via `runWithCache`, so even if the harness cache holds a stub-containing artifact from `TestFixtures` running first, the negative test is hermetic.

Manual toolexec runs against an existing module should keep their own cache: `GOCACHE=$(mktemp -d) go build -toolexec=q ./...`.

**Rebuild the binary after preprocessor changes.** `go install ./cmd/q` (or whatever build command is used) before re-running toolexec builds, or the old binary on `$PATH` will run instead. The e2e harness does this automatically in `TestMain`.

Files to scan after each change:

- `README.md` — if the user-facing surface or smallest example changes.
- `CLAUDE.md` — this file. "Current implementation state" list, conventions.
- `docs/design.md` — if the authoritative design shifts or gains new clauses.

If a commit deletes code or infrastructure that held context, the context it captured must move into a persistent doc in the same commit.

## Naming is not frozen

API identifiers, package boundaries, and file layouts here are chosen to read well at their point of use, not to be permanent. If a better name emerges during implementation or review — clearer reading, less ambiguity, better match to semantics — rename it. Update every reference (code, tests, docs, README snippets) in the same commit, and note the prior name and rationale in the commit message so the change is easy to trace.

The current names came out of an explicit naming round (`chain` → `q`, `Wrap` → `CustErr` → `TryE`, `On*` → `Try*` family, `2`-suffix → `Catch`). Future renames are welcome under the same discipline.
